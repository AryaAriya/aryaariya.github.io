Overview

Many trading card game players struggle with disorganized card collections, unsafe or unreliable trading methods, and limited opportunities for gameplay and community.

The overall goal was to design, code, and create a platform that provides insightful, easy-to-navigate, and accessible content for a wide range of movie lovers, ranging from casual viewers to dedicated cinephiles.

Pain Points

The following are the core pain points shared by all of the primary users that this project aims to take into consideration:

  • Slow or clunky user interfaces lead to user frustration and unsmooth play sessions
  • Unclear or overwhelming tutorials overwhelm and don’t guide both newcomers and returning players
  • Lack of family-friendly or community-driven content lowers deeper engagement and doesn’t encourage shared play
  • Limited language and accessibility support doesn’t make the game and content welcoming to families and diverse players

Competitive Audit

I did a competitive audit on four websites:

Hearthstone

Strengths

  • Strong brand recognition via Warcraft Universe
  • Polished visuals and animations
  • Large community and player base
  • Frequent expansions and content updates
  • Accessible gameplay for casual players

Weaknesses

  • Heavy reliance on monetization (pay-to-win)
  • Progression system feels grind-heavy for casual players
  • Limited deck-building
  • Visually cluttered menus
  • Aging mechanics with fewer innovations

Key Takeaways

Moria Reviews excels in easy navigation, but lacks polish and color, and is rather cluttered. I should prioritize visual polish while maintaining strong structure.

Magic: The Gathering Arena

Strengths

  • Deep strategic complexity and competitive depth
  • Legacy brand with high recognition for decades
  • Integration with physical cards by codes or events
  • High-quality card art and expansive, consistent lore
  • Strong competitive scene

Weaknesses

  • Steep learning curve due to complex rules
  • Resource-heavy client; performance issues on older devices
  • Dense card text and visuals
  • Progression system requires significant time or money
  • Less accessible for casual audiences

Key Takeaways

RogerEbert.com’s site excels in most areas, but the color palette is a bit too bright for my taste. My design should take into consideration the color pallatte and which colors to choose.

Marvel Snap

Strengths

  • Global Marvel IP with iconic, instantly recognizable characters
  • Fast-paced matches (3 minutes), which are suited for mobile
  • Innovative location-based mechanics
  • Highly accessible onboarding and progression
  • Strong engagement through cosmetics and seasonal content

Weaknesses

  • Limited deck-building
  • Heavy reliance on Marvel IP (may overshadow gameplay innovation)
  • Heavy monetization through cosmetics and seasonal gameplay and passes
  • Less robust competitive tracking tools

Key Takeaways

RogerEbert.com’s site excels in most areas, but the color palette is a bit too bright for my taste. My design should take into consideration the color pallatte and which colors to choose.

Slay the Spire

Strengths

  • Unique fusion of roguelike and deck-building mechanics
  • High replayability via procedural generation
  • Lightweight install and stable performance across platforms
  • Well-known among indie players and people who love and use strategy-based methods and games
  • Affordable one-time purchase model

Weaknesses

  • Single-player only; lacks community features
  • Minimalist visuals and designs can be polarizing
  • Sparse lore and limited narrative depth
  • Limited accessibility features
  • Replayability heavily relies on procedural generation, which can feel repetitive

Key Takeaways

RogerEbert.com’s site excels in most areas, but the color palette is a bit too bright for my taste. My design should take into consideration the color pallatte and which colors to choose.

Analysis

Market Gaps From Competive Audit

  • Inclusive design
  • Engaging narrative campaigns alongside competitive play
  • Fair, transparent monetization models that rewards skill and engagement rather than spending
  • Digital play complementing collectible ownership
  • Approachable onboarding combined with meaningful long-term mastery
  • Seamless play across desktop, mobile, and console without any trouble
  • Stronger community features (guilds, trading, co-op modes)

using examples of how I use both Use cases and User journeys. Usually in my projects I will use Use Cases and User Journeys to break down the needs and process of the user. I'm going to use a library for this example. In the first step I will create Use Cases which essentially outline what goals the User has. Use Cases examples: As a library member... I want to take out a book. As a library member... I want to return a book. Now that I have my Use Cases I need to break down these goals into steps, to define the process at every possible stage. So, taking each Use Case I will usually create a flow diagram for that (User Journey). User Journey example: User Journey 1 (based on Use Case 1) User enters the library -> User goes to the historical section -> User finds desired book on WW1 history -> User goes to reception ... etc etc and you follow the flow at each possible step until the Use Case has been fulfilled. The journey will help you identify areas in the process which can be improved.

Oppertunities

  • Combine accessibility with strategic depth
  • Deliver seamless play across desktop, mobile, and console without performance issues
  • Build strong community features
  • Offer cards that exist both digitally and physically
  • Add colorblind mode, text-to-speech, simplified UI, and adaptive difficulty.
